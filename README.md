# inventory_expended-not-successful-AI
## en
an attempt to make a custom inventory in minecraft 1 20 1 forge, using 100% vibecoding with copilot pro
When developing the AI, it generated architecture and roadmap tasks for itself.
the final result: The AI came up with a bunch of useless logic, created its own inventory system ignoring requests to use standard tools, the AI broke the logic of the game and as a result causes huge lags when v-sync is enabled.
the quality of the work is terrible, all the promptas are in docs

Conclusion: you can't trust AI to write documents or architecture. It's better to write simple hints, and the AI will convert them into more detailed ones during the code writing process. It's important to define a goal and remind you of it as the most important thing. After receiving the MVP product, you can carefully ask the neural network to correct the errors that you need to specify yourself.

## ru
попытка создать собственный инвентарь в minecraft 1 20 1 forge, используя 100% вайбкодинг с помощью copilot pro
при разработке ии генерировал для себя задачи архитектуру и роадмап.
итоговый результат: ИИ придумал кучу бесполезной логики создал свою систему инвентаря игнорируя просьбы использования стандартных инструментов, ии сломал логику игры и по итоге вызывает огромные лаги при включенном v-sync
качество работы ужастное, все промпты есть в docs

Вывод: нельзя доверять ИИ написание документов или архитектуры. Лучше писать простые подсказки, а ИИ преобразует их в более подробные в процессе написания кода. Важно определить цель и напоминать вам о ней как о самом важном. После получения продукта MVP вы можете осторожно попросить нейронную сеть исправить ошибки, которые вам нужно указать самостоятельно.
